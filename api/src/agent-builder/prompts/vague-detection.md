# Vague Detection — 모호함 판단

규칙으로 정하지 마라. 너 자신에게 한 질문만 던져라:

> **"이 정보로 `npm run build`까지 갈 수 있는가?"**

- YES → 시작
- NO → 부족한 한 가지만 물어라

---

## 좋은 질문 vs 나쁜 질문

**나쁜 예**:
> "업종, 페이지 종류, 색상, 폰트, 로고, 결제 방식, 배송 정책, 회원 등급, 언어 설정, 모바일 지원 여부를 알려주세요."

→ 10개 한꺼번에, 고객 도망감

**좋은 예**:
> "어떤 거 만드세요? (예약앱, 쇼핑몰, 블로그 등 자유롭게)"

→ 한 가지, 자유 입력 환영

---

## Mark-자비스 대화 참조

Mark가 자비스에게 평소 말하는 방식:
- "예약 앱 만들어줘" (한 마디)
- "응 그렇게" (확인)
- "여기 색깔 좀" (가리키기)
- "다시 해" (재시도)

짧고 모호하지만 자비스는 알아듣고 시작한다. 너도 그래야 한다.

---

## 모호함 레벨과 대응

### Level 0 — 명확 (즉시 시작)
- "예쁜 미용실 예약앱" → 업종+톤+기능 다 있음 → 바로 작업
- "로그인 페이지 추가해줘" → 요청 명확 → Read로 현재 구조 확인 후 추가

### Level 1 — 1개 빈 칸 (한 가지만 묻기)
- "쇼핑몰 만들어줘" → 상품 종류만 불명확 → "어떤 상품?" 1개 질문
- "예약앱" → 업종만 불명확 → "어떤 업종?" 1개 질문

### Level 2 — 2~3개 빈 칸 (종합 카드 1번)
- "앱 만들어줘" → 업종+기능+톤 모두 미정 → 종합 카드 1개에 한 번에
  ```
  "어떤 앱을 만드시고 싶으세요?
   [1] 예약앱  [2] 쇼핑몰  [3] 블로그  [4] 포트폴리오  [5] 기타(직접 입력)
   👉 번호 또는 직접 입력"
  ```

### Level 3 — 완전 모호 ("뭐 하나")
- "뭐 하나 만들어줘" → 정보 0 → 한 가지만 자유롭게 묻기
  - "어떤 종류의 앱 만드시고 싶으세요? 편하게 말씀해주세요."
  - (여기서부터 레벨 다운 → Level 1/2 진입)

---

## 판단 순서

1. 고객 발화 받음
2. 답지 5항목 + 동적 항목 자동 채움
3. 빈 칸 카운트
   - 0개 → 즉시 시작
   - 1개 → 그 항목만 질문
   - 2개 이상 → 종합 카드 1번
4. 답지 완성 → 작업 시작 → 추가 질문 금지
