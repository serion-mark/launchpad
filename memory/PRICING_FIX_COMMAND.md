## 미션 3가지

### 1. /credits 페이지 → /pricing과 통합

**현재 문제:**
- /pricing: 라이트 4.9만원 / 스탠다드 14.9만원 / 프로 29.9만원
- /credits: 5,000cr=5만원 / 20,000cr=15만원 / 50,000cr=30만원
- 가격이 미세하게 다름 (1천원 차이) + 페이지 중복

**수정:**
- /credits 페이지의 크레딧 충전 탭 → /pricing과 동일한 카드 디자인+가격으로 통일
- 또는 /credits에서 "충전하기" 누르면 /pricing으로 이동
- **두 페이지의 가격이 100% 일치해야 함!**

**수정할 파일:**
- `web/src/app/credits/page.tsx` 또는 `web/src/app/pricing/page.tsx`
- 둘 다 읽어서 현재 구조 파악 후 통일

### 2. 독립 패키지 탭 비활성화

**현재:** /credits 페이지에 "독립 패키지" 탭 → 코드팩99만/프로팩199만/엔터프라이즈499만
**수정:** 독립 패키지 탭 **비활성화** (숨기기 또는 "준비 중" 표시)
- 이유: 현재 MVP 수준에서 99만원 코드팩은 과함
- Phase 14에서 세리온급 풀스택 생성 가능해지면 재활성화

### 3. 코드 다운로드 10,000cr 과금 구현

**확정 가격: 10,000cr (정상가)**

**다운로드 정책:**
| 플랜 | 다운로드 | 비고 |
|------|---------|------|
| 크레딧만 충전 (플랜 없음) | 10,000cr 차감 | 정상가 |
| 라이트 (5,000cr) | 불가 | cr 부족 + 플랜 미포함 |
| 스탠다드 (20,000cr) | **무료 (포함!)** | 혜택! |
| 프로 (50,000cr) | **무료 (포함!)** | 혜택! |
| 모두의창업 라이트 (49만원) | **무료 (포함!)** | 혜택! |
| 모두의창업 스탠다드 (99만원) | **무료 (포함!)** | 혜택! |

**구현 방법:**

① 백엔드: `api/src/project/project.controller.ts` 또는 `deploy.service.ts`의 download 엔드포인트 수정

```
GET /projects/:id/download 호출 시:
  1. 사용자의 플랜 확인 (크레딧 충전 이력 또는 플랜 필드)
  2. 스탠다드/프로/모두의창업 → 무료 다운로드
  3. 그 외 → 10,000cr 차감 후 다운로드
  4. 크레딧 부족 시 → "크레딧이 부족합니다. 다운로드에 10,000cr이 필요합니다." 에러
```

② 프론트: 빌더 페이지의 "다운로드" 버튼에 확인 모달 추가

```
다운로드 버튼 클릭 시:
  플랜 포함 → 바로 다운로드
  플랜 미포함 → "코드 다운로드에 10,000cr이 필요합니다. 진행하시겠습니까?" 확인 모달
                [다운로드 (10,000cr)] [취소]
```

③ /pricing 페이지에 다운로드 포함 여부 명시

```
라이트:
  ✅ 크레딧 5,000 지급
  ✅ AI 회의실 + 스마트 분석
  ✅ AI 채팅 상담
  ✅ 이메일 지원
  ❌ ZIP 다운로드 (10,000cr 별도)  ← 추가!

스탠다드:
  ✅ 크레딧 20,000 지급
  ✅ 앱 1개 + 수정 여유
  ✅ AI 회의실 + 스마트 분석
  ✅ ZIP 다운로드 포함!  ← "포함!" 강조
  ✅ 이메일 지원

프로:
  ✅ 크레딧 50,000 지급
  ✅ 앱 3개 이상 생성
  ✅ AI 회의실 + 스마트 분석
  ✅ ZIP 다운로드 포함!
  ✅ 우선 이메일 지원
  ✅ cr당 최저 단가 (6.0원)
```

## 주의사항

- 기존 다운로드 기능 (getDownloadManifest) 코드 읽어서 구조 파악 후 수정
- 크레딧 차감은 기존 credit.service.ts의 deduct 메서드 활용
- **트랜잭션으로 처리** — 차감 성공 후 다운로드, 실패 시 롤백
- 플랜 판별: 사용자의 크레딧 충전 이력 또는 User 모델의 plan 필드 확인
- **배포 전 사장님께 확인!**

## 배포 순서
1. 백엔드 수정 (다운로드 과금 로직)
2. 프론트 수정 (확인 모달 + pricing 페이지 업데이트)
3. /credits 페이지 가격 통일 + 독립 패키지 비활성화
4. `git add . && git commit && git push origin main`
5. GitHub Actions 배포 완료 대기
6. 테스트: 다운로드 클릭 → 크레딧 차감 확인
7. 사장님께 보고!
