## 3. 빌드 파이프라인 현재 구조

```
Step 1: 파일 저장 + ensureNextConfig + ensurePackageJson
Step 2: npm install
Step 2.5: 빌드 전 자동 수정
  - TypeScript 패키지 강제 설치
  - .ts → .tsx 자동 리네임 (JSX 포함 파일)
  - 서버 import → 클라이언트 전환 (next/headers, supabase/server)
  - 'use client' 자동 추가
  - 동적 라우트 디렉토리 삭제
  - middleware.ts 삭제
Step 3: next build 루프 (최대 3회)
  - 매 시도 전 ensureNextConfig (F6 덮어쓰기 방지)
  - F6: 에러 감지 → F4 연계 (잘린 파일 이어쓰기) → AI 수정 → 재빌드
Step 4: out/ → /var/www/apps/{subdomain}/ 복사
Step 5: 소스 정리
```

## 4. Lovable 벤치마킹 인사이트

사장님이 직접 Lovable 사용:
- **한국어 번역 품질 매우 낮음** — "시사"(Preview), "사랑스러운 사람에게 물어보세요"(Ask Lovable) 등
- **하지만 "타이핑 3번만에 앱 화면"은 강력** — 실시간 미리보기가 결제 전환 핵심
- **우리 차별화**: 한국어 네이티브 UX + 정부지원사업 특화 + 코드 다운로드

### 핵심 포지셔닝 (사장님 아이디어)
> "30만 원으로 작동하는 MVP를 만들고, 그걸 들고 외주사 미팅에 가세요. 견적은 반으로, 소통 비용은 제로로."
> "기획서 100장보다, 작동하는 MVP 1개가 가장 완벽한 기획서입니다"

**포지셔닝 5대 장점:**
1. **가격 앵커링**: 외주 3,000만원 vs Foundry 30만원 → 100배 차이
2. **실체가 있는 기획서**: PPT가 아니라 작동하는 앱을 보여주는 것
3. **외주비 협상력**: "이 코드 그대로 디자인만 입혀주세요" → 견적 절감
4. **Lock-in 없음**: Bubble/아임웹과 달리 Next.js 코드 다운로드 → 개발자 인수인계 가능
5. **스택 신뢰도**: Next.js + Supabase = 글로벌 1위 스택

→ 예창패/초창패 창업자에게 완벽한 타겟 메시지. 사업계획서에 이 포지셔닝 반영 필요.

### Lovable "실시간 미리보기" — 결제 전환의 핵심
- 현재 Foundry: 코드 생성 후 "배포하기" 눌러야 결과를 볼 수 있음
- **Lovable**: 코드 생성과 동시에 오른쪽 패널에 실시간으로 앱이 보임
- 사용자가 결제하는 이유:
  1. 코드가 아니라 **"와 이거 진짜 앱 같다!"** 라는 시각적 임팩트
  2. **즉시 미리보기** — 타이핑 3번만에 앱 화면
  3. **"더 만들고 싶다"** → 크레딧 충전
- **반드시 구현해야 할 기능** (Phase 8 사업화의 핵심)

## 5. AI 코드 생성 지침서 (Phase 7 선행 작업)

빌드 에러 11개의 근본 원인 → AI가 코드 생성 규칙을 모르기 때문. 지침서를 프롬프트에 주입하면 에러율 대폭 감소.

### 코드 생성 규칙 (AI가 반드시 따라야 함)
1. `next.config.ts`에 반드시 `output: 'export'` 포함
2. JSX가 있는 파일은 `.tsx` 확장자 사용
3. `createClient()`는 클라이언트용만 사용 (서버용 `cookies()` 사용 금지 — static export 불가)
4. 마크다운 코드블록(```) 사용 금지 — 순수 코드만 출력
5. 곱하기 `*` 사용 시 공백으로 감싸기 (sanitizeCode 안전)

### 검증된 보일러플레이트 (템플릿 코드)
1. **Supabase 클라이언트 설정** — `src/utils/supabase/client.ts`
2. **인증 패턴** — `'use client'` + Supabase Auth
3. **CRUD 패턴** — 목록/생성/수정/삭제 표준 구조

→ 이 지침서를 AI 프롬프트에 주입하면 빌드 성공률 90%+ 기대
