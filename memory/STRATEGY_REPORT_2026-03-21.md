## 1. Foundry 핵심 철학 — "무한한 AI를 일반인의 손에"

### 1.1 사장님 원문 (3/21 대화)

> "AI의 가능성은 무한한데, 일반 사용자는 그 한계를 정해놓고 사용하지.
> 우리는 그 무한한 영역을 연결해주는 도구라고 생각해!"

> "우리의 고객은 AI 전문가가 아니야. AI를 채팅으로 사용하는 사람들을
> 나같은 전문가처럼 만들어주고 돈을 받는 서비스를 하는 거잖아?"

### 1.2 철학 해석

```
┌─────────────────────────────────────────────────┐
│                                                 │
│   AI의 가능성 ━━━━━━━━━━━━━━━━━━━━━━━━━ 무한 ∞  │
│                                                 │
│   일반인이 쓰는 범위 ━━ [아주 작은 원 ○]         │
│   "ChatGPT한테 질문하는 것"                      │
│   "코딩은 안 되겠지", "앱을 만든다고?"            │
│   → 스스로 가능성에 한계를 정해버림               │
│                                                 │
│   Foundry의 역할 ━━━━━━━━━━━━━ [○ → ∞ 확장]     │
│   그 작은 원을 무한대로 확장하는 다리              │
│   질문만 답하면 전문가 수준의 결과물               │
│                                                 │
└─────────────────────────────────────────────────┘
```

### 1.3 이 철학이 사업에 미치는 핵심 영향

**문제**: Foundry는 custom 템플릿 하나로 거의 모든 앱을 만들 수 있다.
하지만 **고객은 그 사실을 모른다.**

- 일반인은 "AI로 앱을 만든다"는 개념 자체가 낯설다
- "아무거나 만들 수 있어요"라고 하면 오히려 뭘 해야 할지 모른다 (선택의 역설)
- 고객은 스스로 가능성에 한계를 정해버린다 ("이런 건 안 되겠지...")

**해결**: Foundry가 해야 할 일은 **가능성을 보여주는 것**이다.

| 방법 | 구현 | 효과 |
|------|------|------|
| 다양한 포트폴리오 | 10개+ 예시 앱 | "이런 것도 만들 수 있구나!" |
| 업종별 템플릿 카드 | 진입 시 카드 선택 | "내 업종이 있네!" → 진입장벽 낮춤 |
| 챗봇 상담 | "이런 앱 되나요?" → "네!" | 가능성의 문을 열어줌 |
| 사용 후기 | "나도 개발 모르는데 만들었어요" | 사회적 증거로 두려움 제거 |

## 2. 포지셔닝 — "같은 걸 더 잘 만들기"가 아니라 "다른 시장 잡기"

### 2.1 경쟁 구도

| | Lovable | Foundry |
|--|---------|---------|
| **시장** | 글로벌, 범용 | **한국, 정부지원사업 창업자** |
| **언어** | 영어 우선 (번역 최악) | **네이티브 한국어** |
| **타겟** | 개발자/디자이너/스타트업 | **개발 모르는 예비창업자** |
| **BM** | SaaS 월정액 ($25~) | **크레딧제 (사업비 정산 가능)** |
| **코드** | Cloud 종속 | **ZIP 다운로드 (100% 소유)** |
| **한국 모듈** | 없음 | **카카오/토스/사업자번호/알림톡** |
| **정부지원사업** | 이해 없음 | **예창패/초창패 맞춤** |

### 2.2 Foundry만의 차별화 — "6가지 Value Proposition"

1. **가격 파괴**: 외주 3,000만원 → Foundry 30만원 (100배 차이)
2. **실체 제공**: PPT가 아닌 **작동하는 앱** → 심사위원 시연 가능
3. **협상력 부여**: "이 코드 그대로 디자인만 입혀주세요" → 외주 견적 절감
4. **자유 보장**: Bubble/Lovable처럼 갇히지 않음 → ZIP 다운로드 → 개발자 인수인계
5. **기술 신뢰**: Next.js + Supabase = 전 세계 개발자 1위 스택
6. **사업비 정산**: 정부지원사업 개발비 항목으로 정산 → **실질적 무료 MVP**

### 2.3 핵심 포지셔닝 메시지

**메인 카피라인:**
> "30만 원으로 작동하는 MVP를 만들고, 그걸 들고 외주사 미팅에 가세요."

**서브 카피라인:**
> "기획서 100장보다, 작동하는 MVP 1개가 가장 완벽한 기획서입니다."

**사장님 철학 반영 카피:**
> "개발자 없이도, 여러분의 사업 아이디어를 작동하는 앱으로 만들어 드립니다."
> "AI가 어렵다고요? 질문에 답하기만 하면 됩니다."

## 3. 타겟 고객 정의

### 3.1 1차 타겟 — 정부지원사업 합격자 (연간 ~13,000명)

| 지원사업 | 선정 규모 | 특징 | 앱 필요 비율 | Foundry 잠재 고객 |
|---------|----------|------|------------|----------------|
| **모두의창업** (지역+테크) | ~3,000명 | 지역기반/테크기반 2트랙 | 30% | ~900명 |
| **예비창업패키지** | ~5,000명 | 사업비 1억, 개발비 배정 多 | 40% | ~2,000명 |
| **초기창업패키지** | ~3,000명 | 3년차 이하, MVP 필요 | 50% | ~1,500명 |
| **재도전 성공패키지** | ~2,000명 | 재창업, 빠른 실행 중시 | 30% | ~600명 |
| **합계** | **~13,000명** | | | **~5,000명** |

### 3.2 고객 페르소나

**페르소나 A: 지역기반 창업자 (모두의창업 지역 트랙)**
- 귀농귀촌 청년, 지역 특산품 판매, 스마트팜 운영
- "우리 농장 앱 만들고 싶은데 외주 견적이 2천만원..."
- 니즈: 상품 목록, 주문, 배송 추적, 농장 소개

**페르소나 B: 테크기반 창업자 (모두의창업 테크 트랙)**
- 헬스케어, O2O 매칭, 소셜앱 등 아이디어 보유
- "앱 만들고 싶은데 코딩을 못 해..."
- 니즈: 프로필, 매칭 알고리즘, 채팅, 결제

**페르소나 C: 외주 견적에 좌절한 예창패 합격자**
- 사업비 1억 중 개발비 3,000만원 배정했지만 견적이 5,000만원
- "PPT로 설명하기 답답한 사람"
- 니즈: 작동하는 MVP → 심사위원 시연 + 외주 협상력

## 4. 벤치마크 앱 분석 (3/21 사장님 제공 5개)

| # | 앱 | 유형 | 핵심 기능 | Foundry 시사점 |
|---|-----|------|----------|---------------|
| 1 | **삐약이** (GLP-1 다이어트) | 테크/헬스케어 | 주사관리, 습관트래킹, 커뮤니티 | → 헬스케어 템플릿 |
| 2 | **나만의그림** (의료/건강) | 테크/O2O | 병원약국찾기, 비대면진료 | → 의료O2O 템플릿 |
| 3 | **인어교주해적단** (수산시장) | 지역/커머스 | 시장별가게, 쿠폰, 기획전, 배송 | → 지역특산품 템플릿, 스마트팜 직판 |
| 4 | **당근마켓류** (중고거래) | 지역/C2C | 동네기반 중고거래, 커뮤니티 | → 동네장터 템플릿 |
| 5 | **숨고** (전문가매칭) | 테크/O2O | AI견적, 카테고리별서비스, **챗봇상담** | → 전문가매칭 템플릿, **챗봇 UI 벤치마크** |

### 벤치마크에서 도출한 인사이트

1. **챗봇 UI가 표준이 되고 있다** — 숨고 우측 하단 "챗봇 상담" 버튼 → Foundry에도 적용
2. **지역기반 앱이 정부지원사업에서 큰 비중** — 인어교주해적단, 당근 스타일 → 템플릿 추가
3. **니치 소셜/매칭이 블루오션** — 시니어, 취미, 동호회 → 포트폴리오에 반영
4. **스마트팜+고객몰 = 지역 트랙 킬러 아이템** — 농장주가 직접 앱 만들어 판매

### 소셜/매칭 앱 — 니치 시장 상세 (사장님 관심 ⭐)

사장님이 특히 좋다고 한 3가지:
1. **취미 기반 매칭** — 등산/러닝/독서/보드게임 등 취미로 사람 연결
2. **지역 동호회 매칭** — 동네 기반 소모임/동호회 가입+활동
3. **시니어 만남** — 5060 세대 친구/동반자 매칭 (블루오션!)

정부지원사업 심사에서 유리한 이유:
- 차별점 명확: "기존 앱은 20~30대 → 시니어용이 없다"
- 심사위원 키워드: "사회적 고립 해소", "시니어 디지털 포용", "지역 커뮤니티 활성화"

Supabase 구현 구조:
```
profiles  — 사진, 소개, 취미, MBTI, 지역
likes     — 좋아요/스킵
matches   — 서로 좋아요 = 매칭
messages  — 1:1 채팅 (Supabase Realtime)
groups    — 동호회/모임
events    — 오프라인 모임 일정
reports   — 신고/차단
```

→ FAQ 자동 응답 (가격, 기능, 소요시간)
→ "지금 만들어볼까요?" → /start 연결
```

### Phase 8 — 고급 기능 + 템플릿 확장

**핵심 원칙: "전문가 수준의 결과물"**

| 작업 | 철학 반영 | 상세 |
|------|----------|------|
| **템플릿 3종 추가** | 가능성 확장 | 지역커머스, 헬스케어, 전문가매칭 |
| **세리온 코드 RAG** | 전문가 수준 코드 | 결제/알림톡/예약 검증된 모듈 |
| **관계형 데이터** | 복잡한 앱도 가능 | JOIN, 중첩 쿼리 |
| **파일 업로드** | 상품 이미지 등 | Supabase Storage |
| **고객 앱 챗봇** | 만든 앱에도 챗봇 | AI or FAQ 챗봇 컴포넌트 자동 포함 |

**템플릿 3종 상세:**
```
① 지역커머스 (local-commerce)
질문지: 판매 품목, 배송/방문, 정기배송 여부, 결제 방식
생성: 상품리스트, 장바구니, 주문, 배송추적, 가게소개

② 헬스케어 (healthcare)
질문지: 관리 대상(복약/운동/식단), 기록 방식, 리마인더
생성: 대시보드, 기록입력, 통계차트, 알림, 커뮤니티

③ 전문가매칭 (matching)
질문지: 서비스 분야, 매칭 기준, 견적 방식, 채팅 필요
생성: 전문가리스트, 견적요청, 매칭, 채팅, 리뷰
```

### 8.1 랜딩 페이지 (page.tsx) 개선 방향

**개선 (일반인 관점):**

         나머지는 AI가 만들어 드립니다."

서브: "개발자 없이, 질문에 답하기만 하면
       작동하는 앱이 완성됩니다."

CTA: "지금 무료로 체험하기" (MVP → 체험)
```

### 8.2 /start 페이지 개선 방향

**현재:**
```
"어떤 서비스를 만드시나요?" → 7개 업종 카드
```

**개선:**
```
"어떤 앱을 만들고 싶으세요?" → 10개+ 업종 카드

카드 구성:
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│ 💇 뷰티  │ │ ☕ 카페  │ │ 🐟 특산품 │ │ 🌿 농장  │
│ 미용     │ │ 음식점  │ │ 산지직송 │ │ 스마트팜 │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐
│ 🏥 헬스  │ │ 🔧 매칭  │ │ 📚 교육  │ │ 💕 소셜  │
│ 케어     │ │ 전문가  │ │ 학원    │ │ 매칭    │
└─────────┘ └─────────┘ └─────────┘ └─────────┘
┌─────────┐ ┌─────────┐
│ 📦 중고  │ │ ✨ 자유  │  ← "자유롭게 만들기"는 맨 마지막
│ 거래    │ │ 롭게    │     (선택의 역설 방지)
└─────────┘ └─────────┘
```

## 9. 타겟별 킬러 메시지 정리

> "우리 동네 특산품 앱, 외주 맡기면 2천만원.
> Foundry로 30만원에 만들고, 나머지는 마케팅에 투자하세요."

### 테크기반 창업자
> "아이디어만 설명하면, 작동하는 MVP가 나옵니다.
> 심사위원에게 PPT 대신 앱을 보여주세요."

> "시니어 매칭앱, 취미 동호회앱...
> 아이디어는 있는데 개발비가 문제? 30만원이면 됩니다."

### 스마트팜/농업 창업자
> "우리 농장 고객 앱, 직접 만들어보세요.
> 상품 등록부터 주문, 배송까지 — 질문에 답하기만 하면 완성."
