## 6. 스마트 분석 프롬프트 반영 + 앱 유형별 필수 구조 (커밋 121c65a)

### 문제
- 소개팅앱 요청해도 미용실 POS가 나옴 (분석 결과 무시)

### 수정 A: 스마트 분석 → 프롬프트 동적 주입
- `projectDescription`에 `[스마트 분석 결과]` 포함 시 → 아키텍처 + 프론트엔드 프롬프트에 자동 주입

- 소개팅 분석 → 틴더/위피 패턴 → 프로필 카드 + 매칭
- 딸기농장 분석 → 마켓컬리 패턴 → 상품 + 장바구니

### 수정 B: 앱 유형별 필수 UI 패턴 (7종)

| 데이팅/소개팅/매칭 | 프로필 카드, 좋아요/패스, 매칭 목록, 1:1 채팅 |
| 커머스/쇼핑몰/판매 | 상품 그리드, 상품 상세, 장바구니, 주문/결제 |

| 커뮤니티/SNS | 피드, 게시물 작성, 댓글/좋아요, 팔로우 |
