## 1. 현재 위치 — 솔직한 진단

### 잘한 것
- **5일 만에 AI MVP 빌더를 만들었다** — Phase 1~5 완료, E2E 배포 성공
- **실제 작동하는 앱을 생성+배포** — https://e2e-sonnet-97dd.foundry.ai.kr
- **자동화 파이프라인** — 코드 생성 → 빌드 → 배포 → 서브도메인까지 원스톱
- **Supabase 자동 프로비저닝** — 고객이 아무것도 안 해도 DB+인증 자동 세팅

### 솔직한 문제점
1. **빌드 성공률이 너무 낮다** — 자동수정 없으면 1/5도 안 됨
2. **AI가 우리 규칙을 모른다** — static export, 클라이언트 전용 등 규칙을 매번 어김
3. **Lovable 대비 UX 격차** — 실시간 미리보기 없음, 결과를 보려면 "배포" 필요
4. **Sonnet 폴백 문제** — smart 요청해도 flash로 떨어지는 경우 발생
5. **동적 라우트 제거** — [id] 페이지가 static export에서 안 됨 → 상세 페이지 없음

## 2. 근본 문제 — "AI가 규칙을 모른다"

이번 E2E에서 발견한 버그 11개 중 **7개가 같은 근본 원인**:
> AI가 "일반적인 Next.js 앱"을 만들지, "Foundry의 static export 앱"을 만들지 모른다

| AI가 한 것 | 우리가 원하는 것 |
|------------|-----------------|
| `cookies()` 사용 (서버) | 클라이언트 Supabase만 사용 |
| `middleware.ts` 생성 | static export에서 불필요 |
| `[id]` 동적 라우트 | static export에서 불가 |
| `.ts`에 JSX | `.tsx` 확장자 필수 |
| `next/headers` import | static export 불가 |
| `output: 'export'` 누락 | 필수 설정 |
| 서버 컴포넌트 사용 | `'use client'` 필수 |

**해결 방향: AI 프롬프트에 "Foundry 코드 생성 규칙서"를 주입**
- 현재: 일반적인 "Next.js 전문가" 시스템 프롬프트
- 개선: "Foundry static export 전용 Next.js 전문가" + 금지 사항 명시 + 보일러플레이트 제공

### 예상 효과
- 빌드 전 자동수정(Step 2.5) 대부분 불필요해짐
- F6 자동수정 횟수 감소 → API 비용 절감
- 빌드 성공률 50% → 90%+ 기대

## 3. 경쟁 분석 재정립 (Lovable 직접 사용 후)

### Lovable의 진짜 강점 (우리가 배울 것)
1. **실시간 미리보기** — 코드 생성과 동시에 오른쪽에 앱이 보임
   → "와!" 하는 순간이 결제 전환점
2. **대화형 수정** — "버튼 색 바꿔줘" → 즉시 반영
3. **즉시 배포** — 버튼 하나로 공유 가능한 URL 생성

### Lovable의 약점 (우리 기회)
1. **한국어 번역 최악** — "사랑스러운 사람에게 물어보세요" ㅋㅋ
2. **한국 비즈니스 모듈 없음** — 카카오 알림톡, 토스 결제, 사업자번호 등
3. **정부지원사업 이해 없음** — 예창패/초창패 맥락을 모름
4. **Lock-in** — Lovable Cloud 종속 vs Foundry는 코드 다운로드

### 결론: "같은 걸 더 잘 만들기"가 아니라 "다른 시장 잡기"
- Lovable = 글로벌, 범용, 영어 우선
- **Foundry = 한국, 정부지원사업 창업자, 외주 대체**

## 4. 포지셔닝 최종 정리

### 타겟 고객
- 예창패/초창패 선정된 예비창업자 (연 10,000+ 명)
- 사업비 중 개발비 배정했지만 외주 견적에 좌절한 사람
- "PPT로 설명하기 답답한" 사람

### 핵심 메시지
> **"30만 원으로 작동하는 MVP를 만들고, 그걸 들고 외주사 미팅에 가세요."**
> **"기획서 100장보다, 작동하는 MVP 1개가 가장 완벽한 기획서입니다."**

### Value Proposition (5가지)
1. **가격**: 외주 3,000만원 → Foundry 30만원 (100배 차이)
2. **실체**: PPT가 아닌 작동하는 앱
3. **협상력**: "이 코드 그대로 디자인만 입혀주세요" → 외주 견적 절감
4. **자유**: Bubble처럼 갇히지 않음 → ZIP 다운로드 → 개발자 인수인계
5. **신뢰**: Next.js + Supabase = 전 세계 개발자 1위 스택

## 6. 이번 대규모 작업에서 배운 교훈

### 개발 프로세스
- **빌드 테스트를 일찍, 자주** — 코드 생성만 하고 빌드 안 하면 문제가 쌓임
- **서버에서 직접 빌드해보기** — 에러 로그가 잘려서 원인 파악이 어려웠음 → 직접 빌드가 가장 빠름
- **에러 로그 길이 충분히** — 2000자 → 4000자로 늘린 것만으로 디버깅 속도 향상
- **자동수정(Step 2.5)은 보험, 프롬프트 개선이 근본 해결**

### 사업 방향
- **경쟁사를 직접 써보는 것이 최고의 벤치마킹** — Lovable 3분 사용이 100시간 분석보다 가치
- **"기능 차이"보다 "경험 차이"** — 실시간 미리보기 하나가 기능 10개보다 중요
- **한국어 현지화는 진짜 차별화** — 번역 품질만으로도 충분히 이길 수 있음
