## 2. 발견된 오류 (Errors)

### 🔴 Critical

| # | 오류 | 상세 | 상태 |
|---|------|------|------|
| E1 | **Anthropic Rate Limit** | 분당 출력 토큰 10,000 제한. 앱 1개 생성에 4단계 × 여러 파일 = 12분+ 소요 | ✅ 2초 딜레이 + 429 재시도로 우회 |
| E2 | **Sonnet/Opus 404 에러** | API 키(sk-ant-api03-...)로 Haiku만 호출 가능. Sonnet/Opus → 404 | ⚠️ 폴백 구현됨, Anthropic 크레딧 충전 필요 |
| E3 | **504 Gateway Timeout** | 코드 생성 12분+ → nginx 기본 300초 초과 | ✅ 600초로 증가 |
| E4 | **JSON 이스케이프 오류** | curl로 직접 JSON 전송 시 특수문자 파싱 실패 | ✅ 파일 기반 전송으로 해결 |

### 🟡 Medium

| # | 오류 | 상세 | 상태 |
|---|------|------|------|
| E5 | **AI 생성 코드에 마크다운 혼입** | TSX 파일에 ```, ###, ✅ 등 마크다운 문법 삽입됨 → 빌드 에러 | ❌ 미해결 |
| E6 | **존재하지 않는 아이콘 import** | AI가 @heroicons/react 등 설치 안 된 패키지 import | ❌ 미해결 |
| E7 | **코드 잘림 (maxTokens 8192)** | 긴 페이지 코드가 중간에 끊김 → 불완전한 파일 | ❌ 미해결 |
| E8 | **미사용 import 잔존** | AI가 생성한 코드에 쓰지 않는 import 문 다수 | ❌ 미해결 |

### 🟢 Low

| # | 오류 | 상세 | 상태 |
|---|------|------|------|
| E9 | **deploy.service.ts 도메인 불일치** | `launchpad.kr` 하드코딩 → `foundry.ai.kr`로 변경 필요 | ❌ 미해결 |

## 3. 구조적 문제점 (Structural Issues)

### S1. 코드 생성 품질 — "세리온급" 달성을 위한 핵심 과제

**현재 수준**: 기본 CRUD + 인증 (로그인/회원가입/DB 조회) → **작동은 함**
**목표 수준**: 세리온처럼 복잡한 비즈니스 로직 (예약관리, 결제, 정산, 알림톡) → **아직 멀음**

| 영역 | 현재 | 세리온 수준 | 갭 |
|------|------|-----------|-----|
| 인증 | Supabase Auth (이메일/비밀번호) | 카카오 OAuth + JWT + 역할별 접근제어 | 높음 |
| DB | 단순 CRUD (select/insert) | 트랜잭션, 관계형 JOIN, 집계 쿼리 | 높음 |
| UI | AI 생성 기본 레이아웃 | 반응형 + 모바일 PWA + 다크모드 + 모달 | 매우 높음 |
| 비즈니스 로직 | 없음 | 예약 슬롯 계산, 부분결제, 크론잡, SSE | 매우 높음 |
| 외부 연동 | 없음 | 050bizcall, NHN 알림톡, KPN 카드단말 | 매우 높음 |
| 배포 | 미완성 (수동) | PM2 + nginx + GitHub Actions 자동배포 | 높음 |

**핵심 인사이트**: 현재 파이프라인은 "프로토타입 빌더"에 가까움. "세리온급 프로그램 빌더"가 되려면 아래 개선이 필수.

### S2. AI 프롬프트 한계

- **단일 파일 생성**: 현재 AI는 파일 단위로 생성 → 파일 간 참조/의존성 불일치
- **컨텍스트 윈도우**: Haiku 200K 토큰이지만, 실제 출력은 maxTokens 8192로 제한
- **비즈니스 로직 부재**: AI가 "결제 시스템", "예약 슬롯 계산" 같은 도메인 로직을 제대로 구현하지 못함
- **에러 처리 미흡**: try-catch, loading state, error boundary 등이 누락되는 경우 많음

### S3. Supabase 제한사항

- **무료 플랜**: 프로젝트 2개 제한 → 고객 3명 이상이면 유료 전환 필수
- **Pro 플랜**: $25/월/프로젝트 → 고객이 늘수록 비용 급증
- **RLS 복잡도**: AI가 복잡한 RLS 정책 (역할 기반, 관계 기반) 정확히 생성하기 어려움
- **Edge Functions**: 서버 사이드 로직 필요 시 → 아직 AI 생성 미지원

## 4. 개선사항 (Improvements)

### 🔴 1순위: Supabase 자동 프로비저닝 (B방식)

**목표**: 고객이 클릭 한 번으로 실제 작동하는 앱을 받는다

```
현재:  앱 생성 → 코드 다운로드 → 고객이 직접 Supabase 세팅 → 수동 SQL 실행 😰
목표:  앱 생성 → 자동 Supabase 프로젝트 생성 → SQL 자동 실행 → .env 자동 주입 → 즉시 작동! 🎉
```

**필요 작업**:
1. Supabase Management API 연동 (Personal Access Token)
2. 프로젝트 자동 생성 (`POST /v1/projects`)
3. SQL 마이그레이션 자동 실행 (`POST /v1/projects/{ref}/database/query`)
4. URL + anon key 자동 추출 → .env.local 주입
5. 무료 플랜 시작 → Pro($25/월) 업그레이드 경로

**예상 공수**: 2~3일

### 🔴 2순위: 코드 품질 자동 검증

**문제**: AI가 생성한 코드가 빌드조차 안 되는 경우 빈번

**해결 방안**:
```
1. 마크다운 혼입 방지
   - AI 프롬프트에 "절대 마크다운 문법 사용 금지" 강화
   - 파서에서 ```, ###, ✅ 등 자동 제거 (post-processing)

2. Import 유효성 검증
   - 생성된 코드의 import 문 스캔
   - package.json에 없는 패키지 → 자동 추가 or 제거
   - 존재하지 않는 파일 참조 → 누락 파일 자동 생성

3. maxTokens 동적 조절
   - 페이지 복잡도에 따라 4096~16384 범위 조정
   - 코드 잘림 감지 (마지막 줄이 불완전) → 이어서 생성

4. 빌드 테스트 자동화
   - 생성 후 `npm run build` 자동 실행
   - 에러 발생 시 → AI에게 에러 로그 전달 → 자동 수정 루프
```

**예상 공수**: 3~4일

### 🟡 3순위: 배포 파이프라인

```
1. Static Export: next export → HTML/CSS/JS → PM2 불필요, 메모리 0MB
2. 와일드카드 서브도메인: *.foundry.ai.kr
3. nginx 동적 라우팅: app-slug.foundry.ai.kr → /var/www/apps/app-slug/
4. 자동 SSL: Let's Encrypt wildcard (DNS-01 challenge)
```

**예상 공수**: 2~3일

### 🟡 4순위: 비동기 파이프라인 (SSE)

**현재**: 12분+ 블로킹 요청 → 504 timeout 위험
**목표**: Server-Sent Events로 단계별 진행상황 실시간 전송

```
클라이언트 ← SSE ← 서버
  "아키텍처 설계 중... (1/4)"
  "DB 스키마 생성 중... (2/4)"
  "프론트엔드 페이지 생성 중... (3/4) - dashboard.tsx 완료"
  "설정 파일 생성 중... (4/4)"
  "✅ 완료! 33개 파일 생성"
```

**예상 공수**: 2일

### 🟢 5순위: 세리온급 기능 생성 역량

**장기 목표** — AI가 아래 수준의 코드를 생성할 수 있어야 함:

```
Phase A: 기본 (현재 달성)
  ✅ 로그인/회원가입 (Supabase Auth)
  ✅ 기본 CRUD (테이블 조회/생성/수정/삭제)
  ✅ 기본 레이아웃 (사이드바 + 헤더 + 메인)

Phase B: 중급 (다음 목표)
  - 역할 기반 접근제어 (OWNER/MANAGER/STAFF)
  - 관계형 데이터 (JOIN, 중첩 쿼리)
  - 폼 유효성 검사 + 에러 처리
  - 반응형 UI (모바일/태블릿/데스크톱)
  - 파일 업로드 (Supabase Storage)

Phase C: 고급 (세리온 수준)
  - 복잡한 비즈니스 로직 (예약 슬롯 계산, 재고 관리)
  - 실시간 기능 (Supabase Realtime)
  - 결제 연동 (Stripe, KPN 등)
  - 외부 API 연동 (Edge Functions)
  - 대시보드 + 통계/분석 (차트)
  - 알림 시스템 (이메일, 알림톡)
  - PWA (오프라인 지원)
```

## 9. 결론

**현재 달성한 것**:
- Supabase 기반 코드 생성 엔진이 **실제 작동하는 앱**을 만들 수 있게 됨 (로그인/회원가입/DB CRUD)
- 펫메이트 E2E 테스트 성공으로 파이프라인 검증 완료
- Rate limit 대응, 504 timeout 해결 등 안정성 확보

**세리온급 달성을 위해 필요한 것**:
1. **자동 프로비저닝** — 고객이 아무것도 설정하지 않아도 앱이 즉시 작동
2. **코드 품질** — 빌드 에러 0%, 마크다운 혼입 0%, 미사용 import 0%
3. **비즈니스 로직** — 단순 CRUD를 넘어 복잡한 도메인 로직 생성 역량
4. **배포 자동화** — Static Export + 서브도메인 자동 매핑

**경쟁사 대비 포지셔닝**:
- Base44 (800억 매각): 노코드 빌더 → 우리는 "AI 풀코드 생성"
- Lovable: UI 중심 → 우리는 "실제 작동하는 풀스택 앱"
- Polsia: 범용 → 우리는 "정부지원사업 창업자 특화"

> **"외주 5,000만원 → 크레딧 20~30만원"** — 이 가치 제안이 현실이 되려면,
> Supabase 자동 프로비저닝이 첫 번째 관문이다.
