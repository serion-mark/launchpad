## 2. 사장님 사용설명서 (이 세션에서 체감한 것)

### "이거 어때?" 해독법
- 대부분 이미 결정한 거. "어때"는 확인 요청.
- 가끔 진짜 의견 구함 → 맥락 봐서 판단.
- 확실하지 않으면 솔직하게 의견 내고 반응 봐.

### 방향 잘못 잡으면 바로 잡아줌
- "아냐아냐" "이게 아니고" → 즉시 방향 전환
- 이전 자비스가 에러 메시지/빈 화면에 집중했는데
  사장님: "진짜 문제는 목업이 나오는 거야!!"
- **사장님이 반박하면 그게 맞을 확률 높음**

### 가격/비용에 민감
- 크레딧 가격을 몇 번이고 물어봄
- API 비용 실측을 직접 함 (Anthropic 잔액 확인!)
- "역마진이면 파산이다!!" → 비용 관리 최우선

### 완성도 > 가격
- "문제는 완성도지 가격이 아냐"
- 30만원 MVP가 진짜 돌아가야 함
- 목업이나 깨진 화면 → 사기 느낌 → 절대 안 됨

### "되는 걸로 가자" 마인드
- 이상적 해결보다 현실적 해결 선호
- LivePreview 개선 vs iframe 배포 URL → 후자 선택

### 실전 테스트를 직접 함
- 직접 앱 만들어보면서 버그 발견
- 스크린샷 찍어서 보여줌
- 고객 관점에서 평가 ("이거 보면 누가 돈 내?")

### 핵심 관점 = 고객 여정

- "10분 기다렸는데 목업이면 누가 돈 내?"
