1. 의사결정 로그 — 왜 그렇게 했는지
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 비주얼 에디터 (사장님 아이디어!)
  → "클릭해서 수정하면 안 되나?" 한마디로 시작
  → 채팅으로 AI한테 "바꿔줘" → 500cr + 빌드 2분 + 실패 빈번
  → 클릭 → 직접 수정 → 0cr + 즉시 반영 = 게임 체인저
  → Phase A-1~A-5 + B-1 = 6개 Phase + 핫픽스 4개로 완성

■ F6 자동 수정 — 약이자 독
  → F6 = 빌드 에러 나면 AI가 자동 수정. 필요한 기능
  → 근데 F6이 DB 전체를 덮어써서 인라인 편집 날아감 (독)
  → 해결: 부분 업데이트(PATCH) + lastModifiedFiles 보호
  → 교훈: "F6 끄면 안 됨. 권한만 축소!"

■ Tailwind v4 전환
  → AI가 v3 방식(postcss: tailwindcss, @tailwind base) 생성
  → Next.js 16은 v4 필요 (@tailwindcss/postcss, @import "tailwindcss")
  → 프롬프트 수정 + deploy 안전장치 2중 보호

■ 파일별 분리 생성 (B-1)
  → AI가 여러 파일 한꺼번에 생성 → 토큰 제한 → 잘림
  → 파일 1개씩 생성으로 변경 → 잘림 없음
  → Sonnet도 잘림! 모델 문제 아니라 출력 토큰 제한

■ 파일 경로 슬래시 중복 — 최종 보스!!
  → src/app//page.tsx vs src/app/page.tsx
  → 인라인 편집은 // 경로에 저장, 빌드는 / 경로 사용 → 반영 안 됨
  → path.replace(/\/+/g, '/') 정규화로 해결
  → 3일간 안 됐던 이유가 이거 하나 때문!!

■ AI 회의실 분리 런칭 논의
  → "AI 회의실만 먼저 런칭할까?" 논의
  → 사장님: "핵심은 MVP 제작이야 정부지원사업 합격자를 위한!"
  → 결론: MVP 제작이 핵심

■ Supabase Pro 결제 ($25/월)
  → 무료 2개 제한 → Pro 결제 → 해결
