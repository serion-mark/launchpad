### AI 수정의 근본 문제 발견
- 제미니가 "handleModify가 없다"고 분석 → 틀렸음
- 실제 코드 확인: 수정 파이프라인 완전히 구현되어 있음
- 진짜 문제: 빌드가 깨지는 것 (Haiku가 전체 맥락 없이 추측)
- 사장님: "해결책 찾기보다 근본적으로 잘못된 건 아닌가?" → 비주얼 에디터로 이어짐

### 미리보기 iframe 차단 해결
- nginx `X-Frame-Options: SAMEORIGIN`이 서브도메인 iframe 차단
- 이전 세션에서 서버에서 직접 고쳤지만, deploy.yml에 원래 값 남아있어서 배포 때마다 원복됨
- 3곳 동시 수정: 서버 nginx + setup-wildcard.sh + deploy.yml
- `Content-Security-Policy: frame-ancestors https://foundry.ai.kr`로 변경

### KPN 카드단말기 — 보류
- FDK_Link CORS 문제: KPN이 해결 안 해줌
- Chrome 확장프로그램 또는 CORS 프록시 .exe로 해결 가능
- 사장님: "일단 보류! 다른 거 먼저"
- 조사 결과는 이 세션에 남겨놓음

### 버전 히스토리 접기
- 기존: 3개 항상 표시 → 미리보기 가림
- 변경: 현재 버전 1줄만 표시 → 클릭 시 펼침

### [배포] → [수정사항 적용] 버튼 변경
- 사장님 결정: 편집할 때마다 자동 재배포 하지 마!

## 3. 삽질 기록 — 이거 안 하면 시간 아낌

### 제미니 분석 맹신 금지
- 제미니가 "handleModify가 없다"고 분석 → 코드 확인하니 있었음
- 교훈: 다른 AI 분석은 참고만. 반드시 코드를 직접 확인해

### CORS vs Private Network Access 혼동
- 처음에 Chrome Private Network Access 문제라고 판단
- Firefox에서도 안 돼서 → 일반 CORS 문제로 재판단
- 근데 KPN이 "CORS 설정 없다"고 해서 → 결국 보류
- 교훈: 문제의 정확한 원인을 좁히기 전에 해결책 제시하지 마

### nginx 설정 원복 문제
- 이전 세션에서 서버에서 직접 nginx 고침 → 다음 배포 때 deploy.yml이 덮어씀
- 교훈: 서버 직접 수정 + 소스코드 수정 둘 다 해야 함
