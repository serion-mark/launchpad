## 🔑 V4 세션 핵심 교훈

### 교훈 1 — V3 4대 실수 반복 방지 완수
V3 후속 자비스가 남긴 4대 실수 리스트:
1. ❌ 크레딧 단가 혼동 → V4 는 가격 언급 최소화, 확인 시 `web/src/app/credits/page.tsx` 참조
2. ❌ 세리온/파운더리 고객 혼동 → 이번 작업 전혀 언급 없음 (세리온 레포와 완전 분리)
3. ❌ 외부 AI 무비판 수용 → 사장님 "제대로 개발한거 맞음?" 지적 시 즉시 자체 점검 실행
4. ❌ Phase 간 영향 분석 생략 → Step 11-B 모달 추가 전 AgentChat → useAgentStream 시그니처 호환성 grep 확인

### 교훈 2 — 사장님 "제대로 개발한거 맞음?" 의 가치
사장님이 단순히 D(푸시) 확인 대신 이 질문을 던진 순간, 자비스는 자체 점검에 들어감:
- tsc 통과 ≠ 런타임 안전
- npm run build 추가 실행 결정 → NestJS 메타 + Next.js SSR 검증 통과
- 만약 사장님이 이 한 마디 안 했다면 즉시 푸시 → 혹시 런타임 에러 시 운영 장애

**→ 사장님 지적 = 가장 강력한 버그 필터. V5 도 같은 정신.**

### 교훈 3 — multi-token 출력의 정확성
Step 0 Vision 실측에서 "레이아웃 40%" 평가가 실패했지만 실제 Vision 출력은 multi-token (`"top-nav | feed | grid"`) 으로 사람 눈 기준 정확. § 14.1 체크리스트에 이걸 반영 ("multi-token 허용") — V3 후속 룰을 더 실용적으로 개선.

### 교훈 4 — 검증 3단계 구조 (tsc → build → runtime)
```
Layer 1: tsc --noEmit       (타입만, ~20초)
Layer 2: npm run build      (NestJS 메타 + Next.js SSR, ~1~2분)
Layer 3: runtime (E2E)      (사장님 실측, ~5분/시나리오)
```

Layer 1 만 하면 20% 안전. Layer 1+2 → 80% 안전. Layer 3 까지 가야 100%. V4 는 Layer 2 까지 완료, Layer 3 는 사장님 실측에 위임.

### 교훈 5 — 공용화 결정은 3번째 재사용 때
ImageUploader 를 AnswerSheetCard 에서 추출할지 고민 시, "ReviewStage + /meeting + AnswerSheetCard 3곳 재사용 예정" 확인 후 즉시 공용화. 2곳만 쓰면 inline copy 가 더 단순할 수 있지만 3곳 = 공용화 임계점.
