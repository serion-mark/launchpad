### 2. Case A (단일 cycle 무한 누적) 여전히 가능
- MAX_BYTES 100K 까지 허용으로 올려놨지만, 단일 cycle 이 100K 초과하면 truncate 불가
- `3f59f2f` 코드 확인: `if (nextCycle >= tail.length) break` → 무한 누적 보호 없음
- 장기 대비: pair 단위 fallback 추가 (사장님 승인 후)

### 3. 비용 사고 방지 (아직 구현 X)
- Agent 세션당 `totalCostUsd` 한계 가드 없음 (예: $3 초과 시 자동 중단)
- 4/19 오후 사장님 한 세션 $1.95 기록 — 더 긴 세션 나오면 폭주 가능
- V3 에서 구현 고려

### 4. 회귀 테스트 (Catch 4, jest)
- sanitize 로직 edge case 많음 — 단위 테스트 있어야 V3, V4 에서 안심 수정
- 우선순위 낮지만 가치 높음
