### 핵심 파일
```
launchpad/api/src/agent-builder/
  ├─ agent-builder.service.ts        ← 수제 루프 (Day 9 에 deprecated 예정)
  ├─ agent-builder-sdk.service.ts    ← ⭐ SDK 기반 (V3 의 주 작업 대상)
  ├─ agent-builder.controller.ts     ← feature flag: AGENT_SDK_ENABLED
  ├─ sdk-message-adapter.ts          ← SDKMessage → AgentStreamEvent 변환
  ├─ sdk-tools.ts                    ← MCP server (provision/deploy/check_build/AskUser)
  ├─ sandbox.service.ts              ← ⭐ createSession(userId, { projectId? })
  ├─ event-translator.service.ts     ← 도구 → 단계 변환 (이슈 #4 에서 연결)
  ├─ prompts/
  │   ├─ agent-core.md               ← ⭐ 이슈 #3 에서 채팅 노이즈 + 번호 제안 룰 추가
  │   ├─ intent-patterns.md
  │   ├─ vague-detection.md
  │   └─ selection-triggers.md
  └─ sdk-poc/                        ← Day 0~5 POC (검증용, 운영 무관)
       poc-day0-z.mjs ~ poc-day5-logparse.mjs
```

### SDK 핵심 옵션 (B안 반영)
```typescript
query({
  prompt,
  options: {
    model: 'claude-sonnet-4-6',
    cwd,                                          // projectId 고정 cwd (B안)
    pathToClaudeCodeExecutable: '...linux-x64/claude',  // musl 회피
    env: process.env,                             // pm2 child env 전달
    permissionMode: 'dontAsk',                    // root 환경 대응 (bypassPermissions 불가)
    allowedTools: [...],
    mcpServers: { foundry: foundryMcp },
    systemPrompt: {
      type: 'preset',
      preset: 'claude_code',
      append: systemAppend,                        // foundry prompt + memory + fallback
      excludeDynamicSections: true,
    },
    settingSources: [],
    ...(finalResumeSessionId ? { resume: finalResumeSessionId } : {}),
  },
})
```

### 3단계 Resume 방어선 (B안)
```
Layer 1 (Primary):   resume: sessionId 직접 시도
Layer 2 (Pre-check): jsonl 실존 확인 → 없으면 Layer 3
Layer 3 (Fallback):  getSessionMessages(sessionId)
                     → 전체 projects 스캔
                     → 메시지 배열 → text 요약 → systemPrompt.append
```

```prisma
model Project {

  agentMessages  Json?      // 수제 루프용 (Day 9 deprecated 예정)
}
```

2. **localStorage 덕분에 `?sdk=1` 자동 유지** → SDK 경로
3. SDK resume 또는 Layer 3 fallback 으로 이전 맥락 복원
4. 추가 요청 처리 → 완료 + 번호 제안

### 비상 롤백
- `?sdk=0` 으로 한 번 접속 → localStorage 삭제 → 수제 루프 복귀
- 또는 서버 env `AGENT_SDK_ENABLED=false`
