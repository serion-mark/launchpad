# 파운더리 (Foundry) — 현재 상태 브리핑 (외부 분석용)

> **작성 시점**: 2026-04-22
> **목적**: 외부 LLM(GPT/Gemini/Grok 등)에게 현재 상태를 전달하여 사업성 분석 / 전략 포지셔닝 판단 지원
> **원칙**: 과장 금지. 검증된 것 vs 미검증 구분.

---

## 1. 한 줄 요약

**파운더리는 한국어 비개발자가 한 줄 아이디어 또는 회의 내용으로 풀스택 웹앱(Next.js + Supabase 기반)을 자동 생성·배포하는 서비스다. 2026-04-20 부터 내부 코드 생성기를 Anthropic Claude Agent SDK(외부 "포비"라는 캐릭터)로 교체하여 운영 중이며, 현재 초기 베타 단계.**

---

## 2. 핵심 변화 — "기존 파운더리" vs "에이전트 포비 탑재 파운더리"

| 항목 | Before (3/18 ~ 4/19) | After (4/20 ~ 현재) |
|---|---|---|
| **코드 생성 엔진** | 자체 구현 "수제 루프" (Anthropic API 수동 호출 + 도구 루프 직접 작성) | **Claude Agent SDK** 공식 SDK 채택 |
| **세션 유지** | 메시지 배열 DB 저장 + 직접 truncate/정리 로직 | SDK 내장 세션 이어가기 기능 |
| **도구 아키텍처** | Custom orchestrator (loop 직접 관리) | **MCP (Model Context Protocol)** 서버 구조. Supabase 연동 + 자동 배포 + 빌드 검증 + 사용자 질문 도구 탑재 |
| **캐시 히트율** | 측정 불가 / 부분 최적화 | **프롬프트 캐싱 자동 최적화 (높은 히트율 유지)** |
| **세션당 비용 (Anthropic API)** | 앱당 약 $2 수준 | **앱당 약 1~2달러** (약 15~40% 절감) |
| **모델** | Claude Sonnet 4.6 | Claude Sonnet 4.6 (동일) |
| **프롬프트** | 12개 섹션의 독자 룰 체계 | **17개 섹션으로 확장된 독자 프롬프트 체계** (영업비밀) |
| **사용자 진입** | 질문지 5~N단계 | **한 줄 입력 → 포비와 3~6턴 대화형 인터뷰 → 확인 스테이지 2분할 UI** |
| **가격** | 앱 1개 생성 = 6,800 cr | 동일 + **상담/수정 3단계 세분화** (0 / 500 / 1,000 / 1,500 cr) + **첫 앱 무료 (free_trial)** |
| **브랜드 캐릭터** | 없음 | **"포비(Foby)"** — Agent 페르소나 |

**중요**: 레거시 `/builder` 경로는 아직 병존 (홈에서 숨김, URL 직접 접근만). 신규 사용자는 `/start` → `/builder/agent` 경로로 유도.

## 3. 기술 스택 개요

### 백엔드
- **NestJS** (Node.js / TypeScript) — REST API + SSE 스트림
- **Prisma** (PostgreSQL 기반 ORM)
- **Claude Agent SDK** (Anthropic 공식) — 핵심 코드 생성 엔진
- **Anthropic 모델**: Claude Sonnet 4.6 (단일 모델 사용)
- **보조 AI**: Google Gemini (회의실) / OpenAI (폴백)
- Passport JWT 인증

### 프론트엔드
- **Next.js (최신 App Router)** + **React 19**
- Tailwind CSS
- 토스페이먼츠 SDK (결제)

### 생성되는 앱의 스택 (포비가 만드는 것)
- **Next.js 15.x** (정적 빌드 / SSG)
- React 19 + Tailwind CSS
- **Supabase** (자동 프로비저닝)
- Pretendard 폰트 + Lucide 아이콘

### 인프라
- 클라우드 서버 2대 (상세 비공개)
- 도메인: `foundry.ai.kr` + 와일드카드 서브도메인
- Nginx + Let's Encrypt SSL
- GitHub Actions 자동 배포 파이프라인
- 앱마다 Supabase 프로젝트 1개씩 자동 생성

### 개발/운영
- Private GitHub 레포 (대표 1인 주 개발)
- 전체 누적 커밋: 380+ (2026-03-18 ~ 현재)
- 프롬프트 엔지니어링 문서: 총 22,500자 규모

## 4. 현재 사용자 플로우 (에이전트 포비 모드)

### 진입점 A — 한 줄 프롬프트 (`/start`)
```
1. foundry.ai.kr/start 진입
2. 한 줄 입력 (예: "반려동물 돌봄 매칭 앱")
3. [시작] 클릭 → 선택 모달:
   - [🚀 바로 만들기] → AI 요약 → 확인 스테이지
   - [💬 포비와 3~6번 상의] → 인터뷰 모드
4. 인터뷰 모드 진입 시:
   - 포비가 상황에 맞게 동적 선택: 정체성/타겟/핵심기능/디자인/차별점/수익/레퍼런스 중 3~6개 질문
   - 번호 카드 UI (타이핑 최소화)
   - 정보 충분하면 조기 종료
5. 확인 스테이지 (ReviewStage) — 2분할 UI:
   - 좌: 정리된 스펙 카드 (앱이름/타겟/핵심기능/디자인/MVP범위)
   - 우: 포비 채팅 — "결제 추가해줘" 같이 자연어로 스펙 실시간 수정
6. [이대로 시작] → Agent 실행
   - 크레딧 차감: 신규 앱 = 6,800cr
   - 평균 10~16분 소요
7. Agent 가 Next.js 앱 생성 (20~30개 파일) + Supabase DB 자동 생성 + 배포
8. 완료 메시지: 실제 배포 URL + 테스트 계정 정보
```

### 진입점 B — AI 회의실 경유 (`/meeting`)
```
1. foundry.ai.kr/meeting 진입 (별도 기능 — 사업 전략 토론)
2. 컨셉/BM/전략 토론 (10~30턴, 유료 300~1,000cr)
3. 토론 끝에 [포비에게 맡기기] 버튼
4. 회의 종합보고서 → AI 요약 → 확인 스테이지
5. 이후 진입점 A와 동일
```

### 진입점 C — 레거시 (숨김)
- URL 직접 접근만 가능, 홈 링크 제거됨

## 5. Agent (포비) 실행 구조 (고수준)

### 핵심 동작
- 사용자 요청 수신 → Agent SDK 가 자동으로 파일 생성 / 편집 / 빌드 / 배포
- 프로젝트별 고정 작업 공간에서 실행 (세션 이어가기 지원)
- Supabase DB 자동 프로비저닝 + 테이블 마이그레이션 + 테스트 계정 생성
- Nginx 자동 설정 + PM2 프로세스 자동 등록 + SSL 자동화

### 과금 분류 (classifyIntent)
사용자 메시지를 자동 분석하여 요금 차등:
- **consultation (0cr)**: 추천/제안/분석/조언/상담 류 → 무료
- **modify_simple (500cr)**: 색상/텍스트/로고 류 단순 수정
- **modify_normal (1,000cr)**: 레이아웃/버튼/간격 류 중간 수정
- **modify_complex (1,500cr)**: 기능 추가/결제/DB/페이지 류 복잡 수정
- **app_generate (6,800cr)**: 신규 앱 생성
- **free_trial (0cr)**: 신규 유저 첫 앱 (UI 노출 예정)

상담 모드에서는 쓰기 도구 자동 차단 (악용 방지).

## 6. 프로덕트 검증 현황

### 검증된 것
- 한 줄 아이디어 → 배포된 웹앱 **End-to-End 파이프라인 작동**
- 고카드 UX 기반 인터뷰 → 스펙 확정 → 자동 생성 플로우 완성
- AI 회의실 → 앱 생성 연계 (내부 검증)
- 앱 수정 모드 (기존 앱에 기능 추가 / 디자인 변경)
- 자동 배포 + 자동 SSL + 자동 Supabase
- 크레딧 기반 차등 과금
- 베타 기간 중 다수의 앱 생성·배포 테스트 완료

### 미검증 / 진행 중
- **시장 규모 / PMF**: 초기 베타 운영 중. 실사용 테스트 단계
- **유료 전환율**: 외부 사용자 유료 전환 사례 없음
- **동시 사용자 스케일**: 부하 테스트 미실시
- **고객 앱의 실제 비즈니스 런칭**: 파운더리로 만든 앱이 실제 서비스로 런칭된 사례 0개

**약점**
- 초기 베타 단계 (외부 사용자 유입 미검증)
- Next.js + Supabase 스택으로 고정 (다른 스택 선택 불가)
- 모바일 앱 생성 불가 (웹 전용)
- 영어 UI 없음 (해외 진출 준비 X)

1. **PMF**: 한국 비개발자가 Next.js+Supabase 기반 웹앱을 AI로 만들고 싶어할까? 어떤 세그먼트(소상공인 / 크리에이터 / 1인 창업자 / 교육)가 가장 먼저 붙을까?
2. **가격 포지셔닝**: 앱당 6,800원 + 월 호스팅 모델이 적정한가? 구독제 전환이 나은가?

6. **지원사업 전략**: 어떤 공고에 어떤 스토리라인으로 붙어야 가장 강력한 메시지가 될까?

## 끝 — 솔직한 한 줄 평

> 파운더리는 **기술적으로는 제대로 동작하는 AI 기반 한국어 웹앱 생성 플랫폼**이지만, **비즈니스적으로는 PMF 검증 단계의 초기 베타**. 앞으로 3~6개월 핵심 과제는 **"초기 100명 유료/활성 사용자 확보"** 이며, 이 시점에서 외부 AI의 가장 큰 기여는 **"어떤 세그먼트에 어떤 메시지와 채널로 붙어야 전환이 나올까"** 에 대한 구체적 액션 제안임.
