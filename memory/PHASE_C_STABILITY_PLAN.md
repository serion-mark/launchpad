■ 왜 하는가
3일간의 전쟁(A-1~A-5 + B-1)으로 비주얼 에디터를 완성했다.
돌아간다. 고객한테 팔 수 있다.
근데 내부 코드는 전쟁터다. AI가 급하게 짠 코드, F6이 덕지덕지 고친 흔적,
하드코딩, 중복 파일, 경로 버그... 이대로 고객 1,000명 받으면 터진다.

비유: 건물은 세웠다. 지금 사람도 들어와 있다.
근데 내진 설계 안 했고, 배관도 아슬아슬하다.
지진 오기 전에 보강해야 한다.

■ 목표
"한 고비를 넘겼으니 그걸 잘 살리자!"
기능 추가 STOP → 안정성 확보 → 그 위에 쌓기

■ 리팩토링 대상 (우선순위)

  1순위: deploy.service.ts (현재 가장 복잡)
  - 현재 상태: F6 + sync + 교정 + 빌드 + 배포가 한 파일에
  - 목표: 기능별 분리
    → BuildService (빌드만)
    → F6Service (자동 수정만)
    → SyncService (DB↔디스크 동기화만)
    → DeployService (배포 오케스트레이션만)
  - 테스트: 각 서비스별 단위 테스트

  2순위: ai.service.ts (코드 생성)
  - 현재 상태: 프롬프트 + 생성 + F4 이어붙이기가 혼재
  - 목표: 프롬프트 분리 + 생성 로직 분리
    → PromptService (프롬프트 관리)
    → CodeGenerationService (코드 생성)
  - 테스트: 생성된 코드 구조 검증

  3순위: project.service.ts (인라인 편집)
  - 현재 상태: CRUD + 인라인 편집 + subdomain
  - 목표: InlineEditService 분리
  - 테스트: 텍스트 치환 정확도 검증

■ 테스트 전략
  - Jest 사용 (NestJS 기본)
  - 단위 테스트: 각 서비스 메서드별
  - 통합 테스트: 인라인 편집 → DB 저장 → 배포 반영 E2E
  - CI/CD: GitHub Actions에 테스트 추가 → 실패하면 배포 차단

■ 리팩토링 전용 프롬프트
  "이 코드는 현재 완벽하게 작동해.
   비즈니스 로직(결과물)은 1%도 건드리지 말고,
   1) 성능 최적화
   2) 중복 코드 제거
   3) 에러 핸들링 강화
   4) 모듈화
   관점에서만 리팩토링해줘.
   바꾼 이유를 주석으로 상세히 달아.
   반드시 테스트 코드도 함께 작성해."

■ 3단계 안전망

  【1】 단위 테스트 (Unit Test)
  - 각 서비스 메서드별 테스트
  - 인라인 편집: 텍스트 치환 정확도 (JSX 4단계 매칭)
  - F6: 부분 업데이트만 되는지 (전체 덮어쓰기 방지)
  - sync: DB↔디스크 일치 검증
  - 경로 정규화: 슬래시 중복 방지

  【2】 통합 테스트 (Integration Test)
  - 앱 생성 → 배포 → 접속 가능한지
  - 인라인 편집 → DB 저장 → 배포 → 반영 확인
  - F6 발동 → 인라인 편집 보존 확인
  - subdomain 중복 체크 → 에러 반환 확인

  【3】 모니터링 + 알림
  - Sentry DSN 설정 (코드는 이미 있음, DSN만 필요)
  - UptimeRobot: foundry.ai.kr/api/health 등록
  - 배포 실패 시 알림 (Slack/카카오톡)
  - API 응답 시간 모니터링

■ CI/CD 파이프라인 강화
  현재: git push → GitHub Actions → SSH → deploy.sh
  목표: git push → 테스트 실행 → 통과해야만 배포

  .github/workflows/deploy.yml에 추가:
  - npm run test (단위 테스트)
  - npm run test:e2e (통합 테스트)
  - 실패 시 배포 중단 + 알림

■ 에러 복구 매뉴얼
  - 고객 "배포 안 돼요" → 프로젝트 ID로 DB 조회 → 원인 파악 → 수정
  - 고객 "수정이 안 반영돼요" → 경로 확인 → 정규화 → 재배포
  - F6 무한 루프 → F6 시도 횟수 제한 (이미 3회) + 실패 시 사용자 안내

      1,000명이 뛰노는 건물의 내진 설계는 인간 시니어의 몫.
      사장님의 미친 속도(AI) + 대기업급 안정성(시니어) = 최강 조합.

■ 채용 기준
  - 10년차 이상 시니어 백엔드/서버 아키텍트
  - NestJS + Prisma + PostgreSQL 경험
  - 프리랜서 또는 파트타임 (Fractional CTO)
  - 주 2~3일, 월 300~500만원 예상

■ R&R (역할 분담) — 매우 중요!!
  사장님 + AI(자비스): 기능 개발, 신규 피처, 고객 대응
  시니어 개발자: 코드 리뷰, 서버 구조 설계, 성능 최적화, 보안 강화

  명확한 지시:
  "기능은 내가 AI랑 다 만들 테니까,
   당신은 이 코드가 동시 접속자 1,000명을 버틸 수 있게
   서버 구조(인프라)만 세팅하고 코드를 다듬어(용접해) 줘."

■ 시니어가 할 일
  1. C-1 문서 기반으로 아키텍처 리뷰
  2. 병목 구간 식별 (DB 쿼리, API 응답 시간)
  3. 캐싱 전략 (Redis 등)
  4. 로드 밸런싱 (PM2 클러스터 → Kubernetes?)
  5. 보안 강화 (SQL Injection, XSS, Rate Limiting)
  6. CI/CD 파이프라인 고도화
  7. 코드 리뷰 프로세스 확립
