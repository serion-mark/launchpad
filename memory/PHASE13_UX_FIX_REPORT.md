## 버그별 상세

### 🔴 버그 1: 미리보기 하드코딩 — 가장 시급!

**현상**: 데이팅앱 만들어도, 커머스 만들어도 항상 "₩1,280,000 / 오늘 예약 12건 / 김지현·서비스A"

**근본 원인 3개 발견:**
1. SSE 완료 후 `setBuildPhase('done')` 하는데 `project.generatedCode`를 안 가져옴
2. `generatedFiles`가 빈 배열 → `showLivePreview = false` → 하드코딩 목업으로 fallback
3. `done` 상태에서도 `previewTemplate` 조건 충족 → 인터랙티브 미리보기 렌더링

**수정 (3곳):**
```
1. generatedFiles memo — done인데 generatedCode 없으면 streamingFiles fallback
2. handleGenerateComplete — 완료 시 프로젝트 데이터 re-fetch (generatedCode 포함)
3. 인터랙티브 미리보기 — buildPhase !== 'done' 조건 추가 (done이면 절대 안 보임)
```

### 🔴 버그 2: 대기 채팅이 프로젝트를 모름

**현상**: "우리가 만드는 앱이 뭐지?" → "죄송합니다 확인할 수 없습니다"

**원인:**
- 프론트: context에 `이름/템플릿/테마`만 (3줄)
- 백엔드: 시스템 프롬프트가 "회의 결과를 참고" — 회의실용이지 빌더용이 아님

**수정:**
- 프론트: context에 프로젝트명, 설명, 질문지 답변, 스마트 분석, 선택 기능, 테마 전부 주입 (11줄)
- 백엔드: `[현재 프로젝트 정보]` 감지 → 빌더 전용 프롬프트 분기
  - "이전 대화를 모른다고 하지 마세요! 위 프로젝트 정보가 있습니다!"

### 🟠 버그 3+4: 채팅 사라짐 + 초기화

**현상**: 아키텍처 단계에서 대화 → 프론트엔드 단계 → 채팅 사라지고 대화 내용 날아감

**수정 (왼쪽 통합):**
1. `sendGeneratingChat()` 함수 추가 — generating 중 왼쪽 채팅으로 AI 상담
2. `sendMessage()` — generating이면 `sendGeneratingChat` 호출
3. 왼쪽 메시지 영역에 추천 질문 4개 표시 (이 앱의 수익모델은? / 타겟 사용자 분석 등)

**효과**: 채팅 1개로 통일, 단계 바뀌어도 히스토리 유지, 혼란 제거
