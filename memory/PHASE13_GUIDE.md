## 8. 빌드 큐 시스템 + 서버 확장 — ⚠️ 공급기업 선정 시 실행 (지금 안 함!)

**지금은 안 함!** 고객 0명인데 서버 올리면 돈 낭비. 선정되면 5분 만에 올림.

**공급기업 선정 즉시 실행할 것:**
1. 서버 업그레이드:
   - 현재: NCP s2-g3a (2vCPU, 8GB) — 월 50,000원
   - 변경: NCP s8-g2 (8vCPU, 32GB, **1TB 디스크**) — 월 352,000원
   - 동시 빌드 5개+ 가능
   - 관리 서버 1대 = 1인 운영에 적합
   - 1개월 테스트 → 트래픽 없으면 다시 내림 (리스크 35만원)
2. 빌드 큐:
   - Bull 또는 Redis 기반 동시 빌드 제한
   - 대기 중이면 "앞에 N개 작업 중, 약 M분 소요" 안내
   - PM2 클러스터 모드 (API 4~8개 인스턴스)
3. SaaS 바우처 선정 시 서버비 정부 지원 가능 (최대 1,000만원)
**파일**: `deploy.sh` + NCP 콘솔 + nginx 설정

## 9. 디스크 관리 + 자동정리 — ⚠️ 공급기업 선정 시 실행

**핵심 전략**: 서버 주문 시 **1TB 디스크**로 잡기 → 복잡한 Object Storage/블록스토리지 이관 로직 불필요!

**왜 심플하게?**:
- 앱 1개 빌드 = 50~200MB
- 1TB면 앱 **5,000개+** 저장 가능
- 초기 1,000명 × 3개 = 3,000개 → 1TB면 충분
- 복잡한 이관 로직 만들 시간에 기능 개발이 낫다

**저장 정책 (사용자에게 안내)**:
- 기본: 30일 미접속 앱 → 빌드 자동 삭제 (코드는 DB에 보존, 재방문 시 자동 재빌드)
- "오래 보관하기" 버튼 추가 → 클릭 시 삭제 대상에서 제외 (호스팅 결제 중인 앱은 자동 보관)
- 안내 문구: "30일간 미접속 앱은 자동 정리됩니다. 코드는 보존되며 재방문 시 자동 복구됩니다."
- **정리 7일 전 이메일 사전 안내 필수!**

**자동 정리 크론잡**:
1. 매일 새벽 4시 실행
2. 30일 미접속 + "오래 보관" 미설정 + 호스팅 미결제 → 빌드 삭제
3. 코드는 DB Json 필드에 보존 (삭제 안 함)
4. 디스크 80% 이상 → 알림 (이메일/슬랙)
5. 재빌드 로직: 사용자 재방문 → DB에서 코드 로드 → next build → 자동 배포

**파일**: 크론잡 + `api/src/deploy/cleanup.service.ts` (신규) + NCP 콘솔
